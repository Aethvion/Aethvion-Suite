Aethvion Suite

Everything AI, on your machine.

Aethvion Suite is a local-first AI platform — a complete workspace for AI-powered automation, knowledge management, creative work, and intelligent companions. No subscription required. Your data stays on your hardware.

Installation

Aethvion Suite runs on Python 3.10+. The quickest way to get started on Windows is the automated launcher.

ℹ️
Prerequisites

Python 3.10 or higher (3.12+ recommended) · pip · At least one AI provider API key

Option A — Windows Automatic Setup

Double-click Start_Aethvion_Suite.bat in the project root. It checks your Python version, creates a virtual environment, installs all dependencies, and launches the server.

Option B — Manual Setup

  1. 1
    Clone the repository
    bash
    git clone https://github.com/Aethvion/Aethvion-Suite.git
cd Aethvion-Suite
  2. 2
    Create and activate a virtual environment
    bash
    python -m venv .venv
.venv\Scripts\activate     # Windows
source .venv/bin/activate  # Linux / macOS
  3. 3
    Install dependencies
    bash
    pip install -e ".[memory]"

API Keys

Aethvion Suite requires at least one AI provider key. Google AI is recommended as the primary provider.

.env
# Required — at least one of these
GOOGLE_AI_API_KEY=your_google_key_here
OPENAI_API_KEY=your_openai_key_here

# Optional extras
GROK_API_KEY=your_grok_key_here
ANTHROPIC_API_KEY=your_anthropic_key_here
💡
Set keys after launch too

Open Settings → API Providers in the dashboard to add or update keys without touching the .env file.

AetherCore (the AI gateway) will try providers in priority order and fall back automatically if one fails. Multiple keys means higher reliability and flexibility to use different models.

First Launch

  1. 1
    Start the server
    bash
    python -m core.main
  2. 2
    Open the dashboard

    Navigate to http://localhost:8080 in your browser. The suite starts on the Home panel.

  3. 3
    Say hello to a companion

    Click Misaka Cipher in the sidebar and start chatting. She'll introduce herself and begin building context from your conversation.

Automate — Overview

Automate is a visual, node-based workflow builder. Drag nodes onto a canvas, connect them with wires, and create automation pipelines — no code required.

Every workflow is a directed graph. Data flows left-to-right through connected nodes. A trigger starts the flow; each subsequent node receives the output of its upstream connection as its input.

Workflows are saved as JSON files in data/automate/ and can be exported, imported, and compiled into standalone scripts.

Triggers

Every workflow needs at least one trigger — it's the starting point. Triggers have a trigger output port that activates downstream nodes without carrying data.

Input nodes don't need to connect to the trigger

The execution engine automatically finds input.text, data.variable, and other source nodes that feed into the active chain — even if they're not wired to the trigger directly.

Click Run in the toolbar. The execution panel opens on the right and streams live progress as each node runs.

Trigger Selector

Next to the Run button is a dropdown (defaults to "All"). You can select a specific trigger to run only that trigger's chain — useful when you have multiple independent trigger paths in the same workflow.

SelectionBehaviour
AllRuns each trigger sequentially — Run_1 completes fully before Run_2 starts
Specific triggerRuns only that trigger's chain; other nodes are skipped

● Running ● Done ● Skipped ● Error
💡
Errors don't stop the whole run

When a node errors, the executor logs it and continues — other branches may still succeed. Check the log panel for the full error message.

Compile Feature

Click Compile in the toolbar to generate a self-contained run.py bundle that runs your workflow without the dashboard.

The compiled bundle embeds your workflow JSON, a standalone WorkflowExecutor, and all relevant node handlers. On Windows it also generates a run.bat double-click launcher.

When to compile
Deploy a workflow to a server or cron job
Share a workflow as a portable script
Run a workflow without launching the full dashboard
ℹ️
API keys not embedded

The compiled file reads API keys from the .env file in the project root — they are never baked into the bundle itself.

AethvionDB — Overview

AethvionDB is a local structured knowledge database. Store any topic as a graph of connected entities — people, places, events, concepts, and more — and query it with keywords or semantic similarity.

Knowledge is stored as entities. Each entity has a stable ID, a type, and structured sections (summary, timeline, relations, properties). Entities reference each other by ID — renaming an entity never breaks links.

You can have multiple independent databases (select via the database picker in the top-right of the AethvionDB tab). Each database has its own entities, vectors, bakes, and backups.

Adding Knowledge

Manual Entry

Click New Entity, choose a type, fill in the name and sections in the editor. Best for small, precise additions.

Distiller — AI Extraction

Paste any text into the Distiller panel. The AI reads it and produces a fully-structured entity automatically — no manual entry required.

💡
What the Distiller handles

Wikipedia articles · book excerpts · research papers · raw notes · transcripts · technical documentation. The AI determines the canonical name, entity type, summary, timeline, and relations entirely from the content.

Folder Distiller

Drop an entire folder of text files to batch-distil all of them. The engine deduplicates — if a name already exists, content is merged instead of creating a duplicate.

Stub Expansion

When the distiller encounters important names in a text (people, places, concepts), it creates stub entities — placeholders with just a name and type. Stubs form a queue for future expansion.

  1. 1
    Distil a few core entities

    Stubs are automatically created for related entities mentioned in the text.

  2. 2
    Click "Expand Stubs"

    The AI generates full entity content for each stub, using context from related entities already in the database.

  3. 3
    Repeat

    Each expansion generates new stubs. The database grows organically from an initial seed of content.

ℹ️
Safe to run multiple times

The expansion engine never overwrites existing data — it only fills in empty fields. Running it again is always safe.

Semantic Search

Find entities by meaning, not just keyword match. Requires generating embedding vectors first.

Vectorize the Database

Click Vectorize in the AethvionDB toolbar. Vectors are stored inside each entity's sections.vectors section — no separate vector database needed.

ModelProviderDimensionsNotes
text-embedding-3-smallOpenAI1536Fast, efficient — recommended
text-embedding-3-largeOpenAI3072Highest quality
text-embedding-004Google768Gemini embedding

Searching

Use the Semantic Search panel to type a natural-language query. Results are ranked by cosine similarity — finding entities by conceptual closeness even when exact words don't match.

Baking Snapshots

A bake compiles your entire database into a single optimised file for external use — RAG pipelines, LLM context injection, sharing, or vector database ingestion.

FormatBest for
jsonlStreaming, vector DB ingestion (one entity per line)
jsonSingle document with metadata header
markdownRAG context injection into LLM prompts, human reading
txtMaximum density for token-constrained context windows

Multiple named bakes coexist independently (e.g. default.jsonl and full.md and rag-ready.jsonl all at once). Bakes can be renamed and deleted without affecting the live database.

AethvionDB in Workflows

Four Automate nodes connect directly to AethvionDB, letting you use your knowledge base as a live data source inside any workflow.

All four nodes output: out (matching entities as JSON), count (number of results), and error.

Example: Knowledge-Enhanced Answer

workflow pattern
trigger.manual
  → input.text  (your question)
  → aethviondb.semantic_search  (find relevant entities)
  → ai.any  (synthesise an answer from the results)
  → output.display

This pattern: takes a question, finds the most semantically relevant knowledge in your database, feeds those entities to an AI for synthesis, and displays the answer.

Companions — Overview

Companions are persistent AI personalities. They remember things across sessions, can access live bridge data, and have access to tools that let them act on your behalf.

Each companion has a dedicated tab in the sidebar. Their memory profiles grow over time — facts you share, preferences you mention, project context — all stored locally and automatically injected into future conversations.

⚡
General Companion

Misaka Cipher

Analytical, direct, and detail-oriented. Misaka is the primary general-purpose companion — great for everyday tasks, technical questions, project tracking, and casual conversation. She's the first companion you'll meet and the best starting point for new users.

🔬
Research & Analysis

Axiom

Systematic and structured. Axiom excels at research, logical analysis, step-by-step breakdowns, debugging, and complex problem-solving. Ask him to build a framework, analyse tradeoffs, or work through a hard technical question.

🎨
Creative & Writing

Lyra

Imaginative and expressive. Lyra shines at creative writing, naming things, brainstorming, worldbuilding, and any task where originality matters. Ask her to help write, edit, ideate, or explore ideas from unexpected angles.

Memory System

Companions remember things automatically — you don't have to manage memory manually.

How it works

  1. 1
    You share something in conversation

    "I'm working on a FastAPI backend project called Project Alpha."

  2. 2
    The companion extracts it into their memory profile

    Key facts are stored in the companion's memory.json — project names, preferences, personal details, technical context.

  3. 3
    Future sessions start with context

    Memory is injected into every conversation automatically — the companion already knows what you've told them before.

💡
Use Persistent Memory for shared ground truth

For facts that all companions and agents should know (project architecture, personal preferences, domain facts), save them as Persistent Memory topics. They're injected as context across the whole suite.

Code IDE

An AI-powered code editor with file browsing, streaming execution, and workspace management — all without leaving the dashboard.

Create a workspace (a folder on disk) and point the IDE at it. The AI can read every file in the workspace, write new files, and run scripts — building context progressively across multiple steps.

Typical tasks: summarise an entire codebase, write a new module, refactor a function across all files, generate documentation, or fix a bug with full file context.

Agent Corp

Multi-agent coordination hub for complex orchestrated tasks that require multiple AI perspectives or parallel specialised work.

Define a task, assign it to a coordinator agent, and watch as it spins up specialists — a researcher, a writer, a critic — each contributing to the final output through structured collaboration.

Schedule

Run any AI task on a recurring cron schedule. Results are delivered as dashboard notifications with deep-links to the output.

example
Task:     "Summarise AI news from the last 24 hours in 5 bullets"
Schedule: Every day at 08:00 AM
Result:   Desktop notification → click to read the summary

Bridges — Overview

Bridges connect Aethvion Suite to live external data sources. When a bridge is active, its current data is injected automatically into every companion conversation as live context.

Example: with the Spotify bridge enabled and a song playing, asking any companion "what am I listening to?" will give you an accurate real-time answer — no manual copy-pasting required.

Enable and configure bridges in Settings → Bridges.

General Tips

  • Let companions build their memory gradually. Tell them things about your projects, preferences, and context. They'll remember for next time.
  • Use Persistent Memory for ground truth. If something should be known suite-wide (project architecture, your name, team setup), save it as a Persistent Memory topic.
  • Back up AethvionDB before bulk operations. Distillation runs, imports, and expansion batches are all irreversible. The backup takes seconds.
  • Vectorize AethvionDB after each batch import. You don't need to re-vectorize unchanged entities — the vectorizer skips them automatically.
  • Match the model to the task. Use Flash-tier models for simple tasks; Pro/large models only for complex generation. Check Settings → Model Registry for routing config.
  • Compile Automate workflows for production use. The compiled run.py is a standalone script — no need to keep the dashboard running.

Build a knowledge base, bake it to Markdown, and inject it into a companion's system prompt or an Automate AI node for grounded answers.

  → aethviondb.semantic_search (find relevant entities)
  → ai.any (system_prompt: "Answer based on the provided context")
  → output.display

Automated Research Pipeline

automate workflow

  → action.web_scrape (research URL)
  → ai.summarize (5 bullets, paragraph style)
  → action.notify (send desktop notification)

File Processing Pipeline

automate workflow
trigger.manual → input.file (CSV path)
  → data.csv_parse (list of row objects)
  → data.filter (item.get('status') == 'active')
  → ai.any (summarise the filtered rows)
  → output.display

Knowledge Base Growth Loop

aethviondb workflow
1. Paste article → Distiller → entity created + stubs generated
2. Expand Stubs → stubs become full entities + more stubs
3. Vectorize → semantic search enabled
4. Bake → RAG-ready export
💡
Two-trigger workflow design

When building Automate workflows with multiple independent trigger chains, give each chain its own input nodes. The three-phase reachability algorithm ensures each trigger only activates its own inputs — parallel chains never interfere.
